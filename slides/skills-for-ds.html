Root Cause

Your Agent Has Amnesia

Each session starts from zero. Here's the fix — three layers of agent memory:

LAYER 1
CLAUDE.md

Long-term memory — preferences, conventions, role. Loaded every session.

LAYER 2
Skills

Specialized knowledge — metric defs, analysis workflows, report templates. Loaded on demand.

LAYER 3
SubAgents

Delegated workers — isolated context for heavy tasks. Keeps main agent clean.

Today's focus: Layer 2 — Skills

Concept

What Is a Skill?

A folder with 3 components. Agent reads SKILL.md; calls scripts on demand; references when needed.

metric-investigation/
  ├── SKILL.md — strategic philosophy: what, why, gotchas
  ├── scripts/
  │  ├── pull_data.py — parameterized SQL runner
  │  └── compute_stats.py — significance testing
  └── references/
     ├── metric_defs.md — team's metric glossary
     └── table_schemas.md — column descriptions, joins

Like a notebook template + shared utility functions + data dictionary — for your agent, not for you.

Comparison

Skills = Depth

  • Add knowledge to the agent's brain
  • Reusable — loaded on demand, session after session
  • Run in-context — white box, you see reasoning

SubAgents = Breadth

  • Add workers — isolated child agents
  • One-shot — spun up, does work, reports back
  • Isolated context — black box, protects main agent

Best pattern: Skill inside SubAgent = 1+1 > 2

Best Practices

5 Rules for Good Skills

01
Gotchas are gold. Every failure becomes a gotcha. This is how institutional knowledge compounds.
02
Agent reasons, code computes. Don't let the LLM do math. Don't let scripts interpret questions.
03
One skill = one job. Chain atomic skills for complex workflows. Don't build god-skills.
04
Description = trigger. Write WHEN to activate, not a summary. This is the ranking signal.
05
Start rough, iterate from usage. A 70% skill that exists beats a perfect skill in your head. Extract from real work, improve through corrections.
